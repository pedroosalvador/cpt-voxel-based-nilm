# CPT Voxel-Based NILM - Project Instructions

## 📚 Project Overview

**Iniciação Científica**: Classificação de Cargas Elétricas usando Representações Volumétricas derivadas da Conservative Power Theory (CPT) e Redes Neurais Profundas 3D.

### Core Research Concept
- **CPT (Conservative Power Theory)**: Decomposição ortogonal da corrente elétrica em 3 componentes fisicamente interpretáveis: ativa, reativa e residual
- **Voxelização 3D**: Transformação dessas 3 componentes em volumes 3D que preservam relações estruturais
- **ResNet3D**: Rede neural convolucional volumétrica para classificação de cargas

### Key Results (Current Best)
- **Acurácia**: 96.16% no conjunto de teste
- **Métricas**: Precisão, Recall, F1-score ≈ 0.96
- **Separabilidade**: Alta entre a maioria das classes

---

## 🔄 Pipeline Padrão de Desenvolvimento

```
Carregamento de Dados 
    ↓
Cálculo da CPT (decomposição tensão/corrente)
    ↓
Normalização [-1, 1]
    ↓
Voxelização (resolução variável, padrão 8×8×8)
    ↓
Treinamento em Rede Neural (modelo: embedded_train)
    ↓
```

### Quando Executar Cada Etapa

- **Validação experimental**: Usar `validate_cpt.py` apenas para estudos de metodologia (NÃO vinculado ao pipeline principal ainda)

## 🧠 Conceitos Críticos - DEVE SABER

### 1. CPT (Conservative Power Theory)
**Arquivo**: `data_processing/cpt_decomposition.py`

**O que faz:**
- Input: Sinais de tensão e corrente
- Output: 3 componentes ortogonais da corrente
- Implementação: Filtros, médias e integrais móveis

**Importante:**
- Esses 3 componentes são a **base** para a voxelização
- Qualidade do CPT = qualidade das features
- Sempre normalizar após CPT antes de voxelizar

### 2. Voxelização
**Arquivo**: `data_processing/voxelization.py`

**Características:**
- Resolução: **Variável** (padrão 8×8×8, mas pode mudar por experimento)
- Normalização: Sempre em **[-1, 1]** antes da voxelização
- Cada eixo = uma das 3 ortogonais da corrente

**Quando ajustar:**
- Aumentar resolução = mais detalhe (maior custo computacional)
- Diminuir resolução = mais generalização

### 3. Modelo Principal: `embedded_train`
**Arquivo**: `models/architectures/embedded_train.py`

- ResNet3D adaptado para processamento volumétrico
- Modelo atual com melhor desempenho
- Usar como baseline para comparações
